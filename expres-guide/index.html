Middleware Avanzado en Express

Funciones que interceptan el ciclo request/response

▼
Un middleware en Express es una función con acceso a (req, res, next) que puede ejecutar código, modificar la petición/respuesta, terminar el ciclo o llamar al siguiente middleware. Se ejecutan en orden de definición.
app.js — Servidor con middleware avanzado
JS app.js 
const express = require('express');
const app = express();

// ─── Middleware integrado ─────────────────────────────────
// Permite recibir JSON en el body de las peticiones
app.use(express.json());

// Permite recibir datos de formularios HTML (URL-encoded)
app.use(express.urlencoded({ extended: true }));

// Sirve archivos estáticos desde la carpeta 'public'
app.use(express.static('public'));

// ─── Middleware personalizado: Logger ─────────────────────
// Registra cada petición con método, ruta y timestamp
const logger = (req, res, next) => {
  const time = new Date().toISOString();
  console.log(`[${time}] ${req.method} ${req.url}`);
  next(); // Cede el control al siguiente middleware
};
app.use(logger);

// ─── Middleware de autenticación (por ruta) ───────────────
// Protege rutas verificando un token en headers
const auth = (req, res, next) => {
  const token = req.headers['authorization'];
  if (!token) return res.status(401).json({ error: 'No autorizado' });
  next();
};

// ─── Middleware de manejo de errores ──────────────────────
// Recibe 4 parámetros: siempre va al FINAL de las rutas
const errorHandler = (err, req, res, next) => {
  console.error(err.stack);
  res.status(500).json({ error: 'Error interno del servidor' });
};

// Ruta protegida con middleware de auth
app.get('/admin', auth, (req, res) => {
  res.send('Panel de administración');
});

app.use(errorHandler); // Registrar al final
app.listen(3000);
⚠ Desventaja del HTML dinámico con template strings: generar HTML con backticks (``) directamente en JS es difícil de mantener, propenso a inyección XSS, no separa lógica de presentación y no tiene soporte de IDE para autocompletado HTML.

Motores de Plantillas

Concepto, comparativa Handlebars / Pug / EJS

▼
Un motor de plantillas (template engine) permite generar HTML dinámico combinando una plantilla estática con datos variables del servidor. Express los soporta mediante app.set('view engine', 'nombre').

Handlebars

Lógica mínima. Sintaxis {{ }}. Ideal para separar totalmente presentación de lógica.

Lógica mínima Semántico Partials

Pug

Sintaxis indentada (sin tags de cierre). Muy compacto. Similar a HAML. Curva de aprendizaje media.

Compacto Sin HTML Rápido

EJS

HTML puro con etiquetas <% %>. Muy cercano a HTML. Fácil de aprender para quien ya sabe HTML.

HTML nativo Flexible Familiar
Instalación y configuración inicial
BASH terminal 
# Instalar los tres motores de plantillas
npm install express-handlebars hbs pug ejs

# Instalar dependencias de utilidad
npm install morgan dotenv

Proyecto con Handlebars (HBS)

Motor de plantillas con sintaxis {{ mustache }}

▼
Handlebars extiende la sintaxis Mustache. Promueve la separación de preocupaciones al limitar la lógica dentro de las plantillas. Soporta partials (componentes reutilizables) y helpers.
JS app-handlebars.js 
const express = require('express');
const hbs     = require('hbs');
const path    = require('path');
const app     = express();

// Configura Handlebars como motor de vistas
app.set('view engine', 'hbs');

// Define la carpeta donde están las vistas (.hbs)
app.set('views', path.join(__dirname, 'views'));

// Registra la carpeta de partials (fragmentos reutilizables)
hbs.registerPartials(path.join(__dirname, 'views/partials'));

// Ruta principal: pasa datos dinámicos a la vista
app.get('/', (req, res) => {
  res.render('index', {
    titulo:    'Lista de Productos',
    productos: [
      { id: 1, nombre: 'Laptop Pro',  precio: 1200 },
      { id: 2, nombre: 'Mouse Inalámbrico', precio: 45 },
      { id: 3, nombre: 'Teclado Mecánico', precio: 150 },
    ]
  });
});

app.listen(3000, () => console.log('Servidor HBS en :3000'));
HBS views/index.hbs 
<!DOCTYPE html>
<html>
<head>
  <!-- Meta para responsividad -->
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>{{ titulo }}</title>
</head>
<body>
  <!-- Partial reutilizable del navbar -->
  {{> navbar}}

  <h1>{{ titulo }}</h1>

  <!-- Iteración con {{#each}} -->
  {{#each productos}}
    <div class="card">
      <h3>{{ nombre }}</h3>
      <p>Precio: ${{ precio }}</p>
    </div>
  {{/each}}
</body>
</html>

Proyecto con Pug

Plantillas con sintaxis indentada sin tags de cierre

▼
Pug (antes Jade) usa indentación en vez de tags de cierre, lo que produce código muy limpio. Es sensible a los espacios (como Python). Permite mixins, herencia de layouts e interpolación de variables con #{var}.
JS app-pug.js 
const express = require('express');
const app     = express();

// Define Pug como motor de vistas
app.set('view engine', 'pug');
app.set('views', './views');

app.get('/usuarios', (req, res) => {
  // Pasa objeto con datos al template
  res.render('usuarios', {
    titulo:   'Usuarios del Sistema',
    usuarios: [
      { nombre: 'Ana García',  rol: 'Admin'   },
      { nombre: 'Carlos Ruiz', rol: 'Editor'  },
      { nombre: 'María López', rol: 'Viewer'  },
    ]
  });
});

app.listen(3001);
PUG views/usuarios.pug 
//- La indentación define la jerarquía (usa espacios, no tabs)
doctype html
html(lang='es')
  head
    //- Responsividad: meta viewport
    meta(name='viewport', content='width=device-width, initial-scale=1')
    title= titulo
    //- CSS inline con variables para tema oscuro/claro
    style.
      body { font-family: sans-serif; padding: 1rem; }
      .card { border: 1px solid #ccc; padding: 1rem; margin: .5rem 0; }
  body
    h1= titulo
    //- Bucle each: itera sobre el array 'usuarios'
    each user in usuarios
      .card

        h3 #{user.nombre}
        p Rol: #{user.rol}
    //- Condicional
    if usuarios.length === 0
      p No hay usuarios registrados.

Proyecto con EJS

HTML enriquecido con etiquetas <% JavaScript %>

▼
EJS (Embedded JavaScript) permite insertar JS directamente dentro de HTML con etiquetas especiales. <% %> ejecuta código, <%= %> imprime valor con escape XSS, <%- %> imprime HTML sin escape.
JS app-ejs.js 
const express = require('express');
const app     = express();

// Configura EJS como motor de vistas
app.set('view engine', 'ejs');
app.set('views', './views');
app.use(express.static('public'));

app.get('/noticias', (req, res) => {
  res.render('noticias', {
    titulo:  'Portal de Noticias',
    fecha:   new Date().toLocaleDateString('es-CO'),
    noticias: [
      { titulo: 'Node.js 22 lanzado',   categoria: 'tech'     },
      { titulo: 'Expo 2025 en Colombia', categoria: 'eventos'  },
      { titulo: 'Nueva IA de código',   categoria: 'tech'     },
    ]
  });
});

app.listen(3002);
EJS views/noticias.ejs 
<!DOCTYPE html>
<html lang="es">
<head>
  <!-- Responsividad -->
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <!-- <%= %> imprime el valor con escape XSS -->
  <title><%= titulo %></title>
</head>
<body>
  <!-- Partial: incluye el header desde otro archivo .ejs -->
  <%- include('partials/header') %>

  <h1><%= titulo %></h1>
  <p>Hoy: <%= fecha %></p>

  <!-- <% %> ejecuta JS sin imprimir -->
  <% noticias.forEach(noticia => { %>
    <article>
      <h2><%= noticia.titulo %></h2>
      <!-- Condicional inline en EJS -->
      <% if (noticia.categoria === 'tech') { %>
        <span class="badge tech">Tecnología</span>
      <% } %>
    </article>
  <% }); %>
</body>
</html>

API RESTful + Validación con Joi

CRUD completo: GET, POST, PUT, DELETE y prueba en Postman

▼
Una API RESTful expone recursos mediante verbos HTTP estándar. Joi es una librería de validación de datos de entrada que permite definir esquemas de validación de forma declarativa para proteger el servidor de datos inválidos.
BASH terminal 
# Instalar Joi para validación de datos
npm install joi
JS routes/productos.js 
const express = require('express');
const Joi     = require('joi');
const router  = express.Router();

// Base de datos simulada en memoria
let productos = [
  { id: 1, nombre: 'Laptop', precio: 1200 },
  { id: 2, nombre: 'Mouse',  precio: 45   },
];

// Esquema Joi: define reglas de validación del body
const schema = Joi.object({
  // nombre: string requerido, mínimo 3 caracteres
  nombre: Joi.string().min(3).required(),
  // precio: número positivo requerido
  precio: Joi.number().positive().required(),
});

// GET /api/productos — Lista todos los productos
router.get('/', (req, res) => {
  res.json(productos);
});

// GET /api/productos/:id — Obtiene un producto por ID
router.get('/:id', (req, res) => {
  const p = productos.find(x => x.id === +req.params.id);
  if (!p) return res.status(404).json({ error: 'No encontrado' });
  res.json(p);
});

// POST /api/productos — Crea un producto (valida con Joi)
router.post('/', (req, res) => {
  // Valida el body con el schema definido
  const { error, value } = schema.validate(req.body);
  if (error) return res.status(400).json({ error: error.details[0].message });
  const nuevo = { id: Date.now(), ...value };
  productos.push(nuevo);
  res.status(201).json(nuevo);
});

// PUT /api/productos/:id — Actualiza un producto existente
router.put('/:id', (req, res) => {
  const idx = productos.findIndex(x => x.id === +req.params.id);
  if (idx === -1) return res.status(404).json({ error: 'No encontrado' });
  const { error, value } = schema.validate(req.body);
  if (error) return res.status(400).json({ error: error.details[0].message });
  productos[idx] = { ...productos[idx], ...value };
  res.json(productos[idx]);
});

// DELETE /api/productos/:id — Elimina un producto
router.delete('/:id', (req, res) => {
  productos = productos.filter(x => x.id !== +req.params.id);
  res.json({ mensaje: 'Producto eliminado' });
});

module.exports = router;
📮 Prueba en Postman: Importa la colección con los 4 endpoints: GET /api/productos, POST con body JSON {"nombre":"Tablet","precio":300}, PUT /api/productos/1 con el objeto actualizado y DELETE /api/productos/2.

Subir Archivos con Multer

Middleware para manejo de multipart/form-data

▼
Multer es un middleware de Express para manejo de archivos subidos vía formularios multipart/form-data. Permite configurar el destino, nombre del archivo, filtros de tipo MIME y límite de tamaño.
BASH terminal 
# Instalar Multer
npm install multer
JS routes/upload.js 
const express = require('express');
const multer  = require('multer');
const path    = require('path');
const router  = express.Router();

// Configuración de almacenamiento en disco
const storage = multer.diskStorage({
  // destination: carpeta donde se guardan los archivos
  destination: (req, file, cb) => cb(null, 'uploads/'),

  // filename: genera nombre único con timestamp
  filename: (req, file, cb) => {
    const nombre = `${Date.now()}-${file.originalname}`;
    cb(null, nombre);
  }
});

// Filtro: solo permite imágenes (MIME type)
const fileFilter = (req, file, cb) => {
  const validos = ['image/jpeg', 'image/png', 'image/webp'];
  if (validos.includes(file.mimetype)) {
    cb(null, true);  // Acepta el archivo
  } else {
    cb(new Error('Solo se permiten imágenes'), false);
  }
};

// Instancia de multer con configuración y límites
const upload = multer({
  storage:    storage,
  fileFilter: fileFilter,
  limits: { fileSize: 5 * 1024 * 1024 } // 5 MB máx
});

// POST /upload — Procesa un solo archivo (campo "imagen")
router.post('/', upload.single('imagen'), (req, res) => {
  if (!req.file) return res.status(400).json({ error: 'Sin archivo' });
  res.json({
    mensaje:  'Archivo subido exitosamente',
    archivo:  req.file.filename,
    tamaño:   `${(req.file.size / 1024).toFixed(2)} KB`,
  });
});

module.exports = router;
💡 Para múltiples archivos: usa upload.array('fotos', 5) para hasta 5 archivos en el mismo campo, o upload.fields([...]) para campos distintos.

Enviar Correos con Nodemailer

SMTP desde Node.js con Gmail o cualquier proveedor

▼
Nodemailer es el módulo más popular para enviar emails desde Node.js. Soporta SMTP, OAuth2, adjuntos, HTML enriquecido y múltiples transportes (Gmail, SendGrid, Mailgun, etc.).
BASH terminal 
# Instalar Nodemailer y dotenv para variables de entorno
npm install nodemailer dotenv
JS services/mailer.js 
const nodemailer = require('nodemailer');
require('dotenv').config(); // Carga variables del archivo .env

// Transporter: conexión al servidor SMTP
// Para Gmail se debe habilitar "Contraseñas de aplicación"
const transporter = nodemailer.createTransport({
  service: 'gmail',          // Proveedor de correo
  auth: {
    user: process.env.EMAIL_USER, // Desde .env
    pass: process.env.EMAIL_PASS, // App Password de Google
  },
});

/**
 * Envía un correo electrónico
 * @param {string} to      - Destinatario
 * @param {string} subject - Asunto del correo
 * @param {string} html    - Cuerpo HTML del mensaje
 */
const sendMail = async (to, subject, html) => {
  // Opciones del mensaje: remitente, destino, asunto y cuerpo
  const mailOptions = {
    from:    `"Mi App" <${process.env.EMAIL_USER}>`,
    to,
    subject,
    html,
  };
  // Envía el mensaje de forma asíncrona con async/await
  const info = await transporter.sendMail(mailOptions);
  return info.messageId;
};

module.exports = { sendMail };
JS routes/contacto.js — Uso del servicio 
const { sendMail } = require('../services/mailer');

router.post('/contacto', async (req, res) => {
  const { nombre, email, mensaje } = req.body;
  try {
    // Llama al servicio de correo con HTML dinámico
    await sendMail(
      email,
      'Confirmación de contacto',
      `<h1>Hola ${nombre}</h1>
       <p>Recibimos tu mensaje: <em>${mensaje}</em></p>`
    );
    res.json({ ok: true, msg: 'Correo enviado' });
  } catch (err) {
    res.status(500).json({ error: 'Error al enviar correo' });
  }
});
⚠ .env nunca va en Git: agrega .env al .gitignore. Usa EMAIL_USER=tucorreo@gmail.com y EMAIL_PASS=apppassword sin comillas.

Despliegue en Heroku

Deploy de Node.js a la nube paso a paso

▼
Heroku es una plataforma PaaS (Platform as a Service) que permite desplegar aplicaciones Node.js directamente desde Git. Detecta automáticamente el tipo de app y ejecuta npm start.
Pasos para desplegar
  • 1Crea el Procfile en la raíz: web: node app.js — indica a Heroku cómo iniciar el servidor
  • 2Usa process.env.PORT en tu app: Heroku asigna un puerto dinámico
  • 3Asegúrate de tener package.json con el script "start": "node app.js"
  • 4Instala Heroku CLI: npm install -g heroku
  • 5Login: heroku login
  • 6Crea la app: heroku create nombre-de-tu-app
  • 7Deploy: git push heroku main
  • 8Variables de entorno: heroku config:set EMAIL_USER=... EMAIL_PASS=...
JS app.js — Puerto dinámico para Heroku 
// Heroku asigna PORT via variable de entorno
// El || 3000 es el fallback para desarrollo local
const PORT = process.env.PORT || 3000;

app.listen(PORT, () => {
  console.log(`Servidor corriendo en puerto ${PORT}`);
});

web: node app.js
