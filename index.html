Plataforma Node.js

CORE Fundamentos de Node.js
10 temas ▼
¿Qué es Node.js?
Entorno de ejecución JavaScript del lado del servidor basado en el motor V8 de Chrome. Usa un modelo de E/S no bloqueante y orientado a eventos, ideal para aplicaciones escalables en tiempo real.
TEORÍA
Consola — uso básico y avanzado
La terminal es la interfaz principal para ejecutar Node.js. Incluye comandos como node archivo.js, redirección de stdin/stdout y variables de entorno.
HERRAM.
Instalación de Node.js
Proceso de instalación mediante nvm (Node Version Manager), que permite gestionar múltiples versiones de Node.js en el mismo sistema de forma sencilla.
SETUP
¿Qué es npm?
Node Package Manager: el gestor de paquetes oficial de Node.js. Permite instalar, publicar y gestionar dependencias de terceros. Comandos: npm install, npm run, npm init.
HERRAM.
GIT básico y GitHub
Sistema de control de versiones distribuido. Git registra cambios en el código; GitHub es la plataforma en la nube para alojar repositorios. Comandos: git init, commit, push, pull.
HERRAM.
package.json y scripts
Archivo de manifiesto del proyecto Node.js. Define el nombre, versión, dependencias, scripts de ejecución y metadatos. Los scripts permiten automatizar tareas como build, test y start.
CONFIG
Nodemon
Herramienta de desarrollo que reinicia automáticamente el servidor Node.js cada vez que detecta cambios en los archivos del proyecto. Evita reiniciar manualmente en cada edición.
DEV
REPL de Node.js
Read-Eval-Print-Loop: entorno interactivo que permite ejecutar código JavaScript directamente en la consola para pruebas rápidas y exploración de la API de Node.
DEV
Global Objects: CommonJS
__dirname y __filename revelan rutas absolutas del archivo actual. require() importa módulos y exports expone funciones/valores al exterior del módulo.
MÓDULOS

02

JavaScript Moderno en Servidores

ES5+ Callbacks y manejo de errores
5 temas ▼
Callbacks
Función pasada como argumento a otra función para ejecutarse después de que una operación asíncrona termine. Patrón fundamental antes de las Promises.
ASYNC
forEach, for in, for of
forEach itera arrays; for…in itera claves de objetos; for…of itera valores de iterables (arrays, strings, Maps). Cada uno tiene su caso de uso específico.
ITER.
try / catch y throw
Mecanismo de manejo de errores síncronos. throw lanza una excepción; try/catch captura errores para evitar que el programa se detenga inesperadamente.
ERRORES
ES6-11 JavaScript moderno — ES6 al ES11
15 temas ▼
let, const, template strings, arrow functions
ES6 introduce let/const con scope de bloque, template literals con ${}, funciones flecha => con this léxico, clases, Map, filter, spread ... y rest params.
ES6
Promises
Objeto que representa el resultado eventual (éxito o fallo) de una operación asíncrona. Encadena operaciones con .then() y maneja errores con .catch().
ASYNC
async / await (ES8)
Sintaxis sobre Promises que permite escribir código asíncrono de forma secuencial y legible. async marca una función como asíncrona; await pausa la ejecución hasta resolver la Promise.
ES8
Optional chaining y Nullish Coalescing (ES11)
obj?.prop accede a propiedades sin lanzar error si el objeto es null/undefined. ?? retorna el valor de la derecha solo si el de la izquierda es null o undefined.
ES11
Promise.allSettled (ES11)
Espera a que todas las promesas se resuelvan (ya sea con éxito o rechazo) y retorna un array con el estado y valor/razón de cada una. Útil cuando quieres resultados independientemente de fallos.
ES11
TypeScript
Superset tipado de JavaScript que añade tipos estáticos, interfaces y decoradores. Compila a JavaScript estándar. Mejora la detección de errores en tiempo de desarrollo.
TS
RxJS
Librería de programación reactiva basada en Observables. Permite componer operaciones asíncronas y basadas en eventos con operadores como map, filter, mergeMap.
LIB

03

Servidores HTTP en Node.js y Express

HTTP Módulo nativo HTTP y File System
6 temas ▼
Servidor con módulo nativo HTTP
Node.js incluye el módulo http que permite crear servidores sin dependencias externas. Se parsean rutas y métodos manualmente con req.url y req.method.
NATIVO
Postman — pruebas de API
Herramienta GUI para probar endpoints HTTP. Permite enviar requests GET, POST, PUT, DELETE con headers y body personalizados, y visualizar respuestas de forma organizada.
TEST
File System (fs)
Módulo nativo de Node.js para operar con el sistema de archivos. Ofrece métodos síncronos (readFileSync) y asíncronos (readFile) para leer, escribir, copiar y eliminar archivos.
FS
EXPRESS Express.js — Framework Web
5 temas ▼
¿Qué es Express?
Framework minimalista y flexible para Node.js que simplifica la creación de servidores web y APIs REST. Añade un sistema de routing, middlewares y manejo de errores sobre el módulo nativo HTTP.
MARCO
Rutas, listen, headers y errores
Express define rutas con app.get(), app.post(), etc. app.listen(port) inicia el servidor. Los headers se manipulan con res.set() y los errores con middleware de 4 parámetros.
RUTAS
Archivos estáticos en Express
express.static() sirve archivos HTML, CSS, JS e imágenes directamente desde una carpeta pública sin necesidad de definir rutas individuales para cada recurso.
STATIC

04

Uso Avanzado de Middleware Express

VIEWS Motores de plantillas y API REST
8 temas ▼
Motores de plantillas (Handlebars, Pug, EJS)
Sistemas para generar HTML dinámico en el servidor. Handlebars usa {{ }}; Pug usa indentación; EJS mezcla HTML con <%= %>. Se configuran con app.set('view engine').
VIEWS
Heroku — deploy en la nube
Plataforma PaaS (Platform as a Service) que permite desplegar aplicaciones Node.js en la nube con un simple git push heroku main. Gestiona servidores, escalado y dominios automáticamente.
DEPLOY
API RESTful básica (GET, POST, PUT, DELETE)
Arquitectura para APIs HTTP que usa verbos HTTP para operaciones CRUD. GET lee, POST crea, PUT actualiza, DELETE elimina. Las rutas representan recursos como /api/users/:id.
REST
Joi — validación de datos
Librería de validación de esquemas para JavaScript. Permite definir reglas estrictas para validar datos entrantes (query params, body) antes de procesarlos. Retorna errores descriptivos automáticamente.
VALID.
Multer — subida de archivos
Middleware para manejar multipart/form-data, el formato usado para subir archivos. Gestiona la ubicación, nombre y tamaño de archivos subidos al servidor.
FILES
Nodemailer — envío de correos
Librería para enviar emails desde Node.js usando SMTP. Soporta HTML, adjuntos y múltiples proveedores como Gmail, Outlook o servidores SMTP propios.
EMAIL

Bases de Datos Modernas

MONGO MongoDB y Mongoose
9 temas ▼
¿Qué es MongoDB?
Base de datos NoSQL orientada a documentos. Almacena datos en formato BSON (similar a JSON) en colecciones. Flexible, sin esquema fijo, ideal para datos jerárquicos y aplicaciones modernas.
NOSQL
MongoDB Shell — CRUD en consola
CLI oficial para interactuar con MongoDB. Comandos: db.collection.find(), insertOne(), updateOne(), deleteOne(). El lenguaje de query usa objetos JSON como filtros.
CRUD
MongoDB Compass y Robo3T
Clientes GUI para explorar y administrar bases de datos MongoDB. Compass es la herramienta oficial con visualización de esquemas; Robo3T (ahora Studio3T) es una alternativa popular con soporte a scripts.
GUI
MongoDB Atlas
Servicio de base de datos en la nube (DBaaS) de MongoDB. Ofrece clusters gestionados, backups automáticos, monitoreo y conexión desde cualquier lugar con una URI de conexión.
CLOUD
Mongoose — ODM para MongoDB
Object Document Mapper que añade esquemas, validaciones y métodos a los modelos de MongoDB. Define la estructura de los documentos con Schema y crea modelos con mongoose.model().
ODM

06

Sesiones, Storages y Auth

AUTH Passport, JWT y CORS
7 temas ▼
Cookies y sesiones en Express
Las cookies almacenan datos pequeños en el cliente; las sesiones guardan el estado del usuario en el servidor. express-session gestiona sesiones con ID almacenado en cookie.
SESSION
Storages de sesión
Los datos de sesión pueden almacenarse en: memoria (solo dev), archivo (simple), Redis (alta velocidad y escalable) o MongoDB (persistente con connect-mongo).
STORAGE
Passport.js — autenticación
Middleware de autenticación flexible para Node.js. Soporta estrategias como Local (usuario/contraseña), OAuth con Google, Facebook, GitHub y cientos más mediante plugins.
AUTH
JSON Web Token (JWT)
Estándar abierto (RFC 7519) para transmitir información entre partes como un objeto JSON firmado digitalmente. Compuesto por header, payload y signature. Se usa para autenticación stateless en APIs REST.
JWT
CORS en servidores
Cross-Origin Resource Sharing: política de seguridad del navegador que restringe peticiones HTTP entre orígenes distintos. El paquete cors en Express configura los headers necesarios para permitir o restringir acceso.
SEGUR.

WebSockets

WS Socket.io y aplicaciones en tiempo real
6 temas ▼
¿Qué es un WebSocket?
Protocolo de comunicación bidireccional y persistente sobre una sola conexión TCP. A diferencia de HTTP (request-response), WebSocket permite que servidor y cliente se envíen mensajes en cualquier momento.
PROTO.
Socket.io
Librería que simplifica WebSockets con fallbacks automáticos, rooms, namespaces y broadcasting. En el servidor: io.on('connection', socket => {}). En el cliente: socket.emit('evento', data).
LIB
Chat en tiempo real
Aplicación práctica de WebSockets donde múltiples clientes se conectan al servidor y se transmiten mensajes instantáneamente entre sí mediante eventos de Socket.io.
PROYECTO
Paint colaborativo
Aplicación donde múltiples usuarios dibujan simultáneamente en un canvas. Cada trazo se transmite via WebSocket a todos los clientes conectados para visualización en tiempo real.
PROYECTO

Estructura y Herramientas de Desarrollo

ARCH Patrones y herramientas de compilación
8 temas ▼
Patrón MVC — capas del servidor
Model-View-Controller separa responsabilidades: el Modelo maneja datos, la Vista la presentación y el Controlador la lógica. Aplicado al backend: routers, servicios y repositorios.
PATRÓN
DAO / DTO / Repository
DAO (Data Access Object) abstrae las operaciones de DB; DTO (Data Transfer Object) define la forma de los datos entre capas; Repository centraliza la lógica de acceso a datos.
PATRÓN
dotenv — configuración del servidor
Librería que carga variables de entorno desde un archivo .env al objeto process.env. Permite separar configuración sensible (credenciales, puertos) del código fuente.
CONFIG
Babel — transpilación JS
Compilador de JavaScript que transforma código moderno (ES6+) a versiones compatibles con entornos más antiguos. Permite usar import/export de ES Modules en Node.js antes de soporte nativo.
BUILD
Webpack — bundler de módulos
Empaquetador de módulos que procesa assets (JS, CSS, imágenes) y genera bundles optimizados. En backend se usa para dividir código, tree-shaking y gestión de dependencias complejas.
BUILD

Testing

TEST Jest, Mocha y pruebas de API
8 temas ▼
TDD y BDD
TDD (Test Driven Development): escribir el test antes del código. BDD (Behavior Driven Development): describe el comportamiento esperado en lenguaje natural con describe/it para tests legibles.
METODOL.
Jest y Mocha
Jest: framework de testing todo-en-uno con mocks, cobertura y snapshots integrados. Mocha: framework flexible y modular que requiere librerías adicionales como Chai para assertions.
LIB
Pruebas de API REST con Mocha
Uso de supertest para enviar requests HTTP a la API dentro del entorno de tests. Permite verificar status codes, headers y body de las respuestas sin levantar un servidor real.
API
Mocks y Faker.js
Un mock simula el comportamiento de dependencias externas (DB, APIs) en tests. Faker.js genera datos falsos pero realistas (nombres, emails, fechas) para poblar tests automáticamente.
MOCKS

Frameworks y Entornos Backend

FW Adonis, NestJS, Sails, Koa y Deno
6 temas ▼
AdonisJS
Framework MVC completo para Node.js con ORM propio (Lucid), sistema de autenticación, CLI robusto y estructura opinada similar a Laravel. Ideal para aplicaciones empresariales grandes.
FW
NestJS
Framework progresivo para Node.js con TypeScript como primera clase. Usa decoradores e inyección de dependencias similar a Angular. Arquitectura modular muy usada en aplicaciones enterprise.
FW
Sails.js
Framework MVC con soporte a WebSockets integrado y ORM (Waterline) que conecta con múltiples bases de datos. Sigue convenciones sobre configuración para generar APIs rápidamente.
FW
Koa
Framework minimalista creado por los autores de Express. Usa async/await de forma nativa y un modelo de middleware basado en cascada. Muy liviano y sin funciones incorporadas innecesarias.
FW
Deno — el futuro
Entorno de ejecución para JavaScript/TypeScript creado por el mismo creador de Node.js. Seguro por defecto (sin acceso a FS/red sin permisos), soporte nativo a TypeScript y gestión de módulos por URL.
