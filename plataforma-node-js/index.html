01

¿Qué es Node.js?

Definición: Node.js es un entorno de ejecución de JavaScript en el servidor, construido sobre el motor V8 de Chrome. Permite correr JS fuera del navegador, orientado a I/O no bloqueante y alta concurrencia.

⚙️ Arquitectura

  • Motor V8 — Compila JS a código máquina (Google).
  • libuv — Maneja el Event Loop y operaciones asíncronas.
  • Event Loop — Bucle que procesa callbacks sin bloquear el hilo.
  • Single-threaded — Un hilo principal; I/O delegado a workers.
  • Non-blocking I/O — Las operaciones no detienen la ejecución.

🔄 Ciclo de vida

Odd releases (e.g. v21)

Versiones de corta vida — nuevas features, en experimentación.

Even releases → LTS

Long Term Support: 30 meses de soporte activo. Recomendadas para producción.

End of Life (EOL)

Sin parches ni seguridad. Migrar es obligatorio.

Consola — Uso básico y avanzado

La consola (terminal / shell) es la interfaz de texto para ejecutar comandos del sistema operativo. En Node.js es la puerta de entrada a todo.

🖥️ Comandos básicos (bash/cmd)

terminal 
$ ls          # listar archivos (dir en Windows)
$ pwd         # directorio actual
$ cd proyecto # cambiar directorio
$ mkdir src   # crear carpeta
$ touch app.js# crear archivo
$ clear       # limpiar pantalla

⚡ Uso avanzado

terminal 
$ cat app.js          # ver contenido
$ grep -r "require"   # buscar en archivos
$ rm -rf node_modules# borrar recursivo
$ cp app.js backup.js# copiar archivo
$ node app.js         # ejecutar script
$ node -v             # versión de Node

Instalación de Node.js

📥 Métodos de instalación

  • nodejs.org — Descarga directa del instalador oficial (Windows/Mac).
  • nvm — Node Version Manager: administra múltiples versiones.
  • Homebrew (Mac) — brew install node
  • apt (Linux) — sudo apt install nodejs npm

✅ Verificar instalación

terminal 
$ node --version
  v20.11.1

$ npm --version
  10.2.4

$ which node
  /usr/local/bin/node
Recomendación: Usa siempre la versión LTS (Even) en proyectos de producción. La versión Current es ideal para explorar features nuevas.
04

¿Qué es npm?

npm (Node Package Manager) es el gestor de paquetes oficial de Node.js. Permite instalar, actualizar y publicar librerías reutilizables desde el registro público npmjs.com.

📦 Instalación de dependencias

terminal 
# Instalar paquete (producción)
$ npm install express

# Instalar como dependencia de desarrollo
$ npm install nodemon --save-dev

# Instalar globalmente
$ npm install -g nodemon

# Instalar todo (desde package.json)
$ npm install

# Desinstalar
$ npm uninstall express

🧰 Modos de uso npm

  • npm init — Inicializa un proyecto nuevo.
  • npm run — Ejecuta scripts definidos en package.json.
  • npm publish — Publica un paquete al registro.
  • npm audit — Revisa vulnerabilidades.
  • npm outdated — Lista paquetes desactualizados.
  • npx — Ejecuta binarios sin instalar globalmente.

GIT básico y GitHub

GIT es un sistema de control de versiones distribuido. GitHub es la plataforma en la nube para alojar repositorios Git y colaborar con otros desarrolladores.

🔧 Flujo básico de GIT

terminal 
# Configuración inicial
$ git config --global user.name  "Tu Nombre"
$ git config --global user.email "tu@email.com"

# Inicializar repo
$ git init

# Ver estado de archivos
$ git status

# Preparar cambios (staging)
$ git add .

# Guardar snapshot (commit)
$ git commit -m "feat: inicializar proyecto Node.js"

# Conectar con GitHub y subir
$ git remote add origin https://github.com/usuario/repo.git
$ git push -u origin main

# Clonar repositorio existente
$ git clone https://github.com/usuario/repo.git
Tip: Crea siempre un archivo .gitignore y añade node_modules/ para no subir las dependencias al repositorio.

🔁 REPL (Read-Eval-Print Loop)

El REPL es un intérprete interactivo de Node.js — ideal para pruebas rápidas.

REPL 
$ node
> 2 + 2
  4
> Math.sqrt(16)
  4
> const x = 'Hola Node'
  undefined
> x.toUpperCase()
  'HOLA NODE'
> .exit   // salir del REPL
07

Inicialización del proyecto · package.json · Nodemon

🚀 Inicializar proyecto

terminal 
# Crea package.json de forma interactiva
$ npm init

# Crea package.json con valores por defecto (más rápido)
$ npm init -y

📄 Anatomía del package.json

json — package.json 
{
  "name":        "mi-proyecto",      // nombre del paquete
  "version":     "1.0.0",            // versión semántica (semver)
  "description": "Proyecto Node.js", // descripción breve
  "main":        "index.js",         // punto de entrada
  "scripts": {                        // comandos ejecutables con npm run
    "start": "node index.js",       // npm start
    "dev":   "nodemon index.js",    // npm run dev (desarrollo)
    "test":  "echo 'no hay tests'"  // npm test
  },
  "dependencies": {                   // necesarias en producción
    "express": "^4.18.2"
  },
  "devDependencies": {               // solo en desarrollo
    "nodemon": "^3.0.1"
  },
  "author":  "Tu Nombre",           // autor del proyecto
  "license": "MIT"                  // licencia open-source
}

👁️ Nodemon — Recarga automática

Nodemon observa los archivos del proyecto y reinicia automáticamente el servidor cada vez que detecta un cambio. Esencial en desarrollo para no ejecutar node app.js manualmente a cada momento.

terminal 
# Instalar como dependencia de desarrollo
$ npm install nodemon --save-dev

# Usar directamente
$ npx nodemon index.js

# O mediante el script en package.json
$ npm run dev

# Salida esperada:
  [nodemon] watching path(s): *.*
  [nodemon] starting `node index.js`
  Servidor corriendo en http://localhost:3000

Global Objects — CommonJS

Node.js provee objetos globales disponibles en cualquier módulo sin necesidad de importarlos. El sistema de módulos CommonJS usa require / exports para compartir código entre archivos.

📁 __dirname

Ruta absoluta del directorio donde se encuentra el archivo actual. Útil para construir paths seguros.

app.js 
// __dirname: ruta del directorio del archivo
console.log(__dirname);
// → /home/usuario/mi-proyecto

const path = require('path');
const ruta = path.join(__dirname, 'data', 'users.json');
// → /home/usuario/mi-proyecto/data/users.json

📄 __filename

Ruta absoluta del archivo actual en ejecución, incluyendo el nombre del archivo.

app.js 
// __filename: ruta completa del archivo actual
console.log(__filename);
// → /home/usuario/mi-proyecto/app.js

📤 exports / module.exports

Objeto que expone funciones o valores de un módulo para que otros archivos los usen con require.

math.js 
// Exportar funciones del módulo
const sumar = (a, b) => a + b;
const restar = (a, b) => a - b;

// Forma 1: module.exports (reemplaza el objeto)
module.exports = { sumar, restar };

// Forma 2: exports (añade propiedades)
exports.multiplicar = (a, b) => a * b;

📥 require (CommonJS)

Función global para importar módulos: propios, de Node.js (built-in) o de npm.

index.js 
// Módulo built-in de Node.js
const fs   = require('fs');
const path = require('path');
const http = require('http');

// Módulo de npm (instalado previamente)
const express = require('express');

// Módulo propio (ruta relativa)
const { sumar } = require('./math');

console.log(sumar(3, 4)); // 7

🏗️ Ejemplo completo — mini servidor HTTP

index.js — Servidor Node.js nativo 
// 1. Importar módulo http nativo de Node.js
const http = require('http');
const path = require('path');

// 2. Global objects para saber dónde estamos
console.log('Directorio:', __dirname);
console.log('Archivo:    ', __filename);

// 3. Definir puerto (variables de entorno o default)
const PORT = process.env.PORT || 3000;

// 4. Crear servidor HTTP
const server = http.createServer((req, res) => {
  res.writeHead(200, { 'Content-Type': 'text/plain' });
  res.end('¡Hola desde Node.js!\n');
});

// 5. Poner el servidor a escuchar
server.listen(PORT, () => {
  console.log(`Servidor en http://localhost:${PORT}`);
});
